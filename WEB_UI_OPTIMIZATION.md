# Web UI 优化方案

## 当前问题
1. ❌ 脚本列表无法点击执行
2. ❌ 显示格式不直观（纯文本列表）
3. ❌ 缺少描述信息
4. ❌ 没有分类展示
5. ❌ 缺少参数输入界面

## 优化方案

### 阶段1：最小化改进（推荐）
**目标：快速提升可用性**

1. **Dashboard 改进**
   - 按站点分组显示（使用 HTML details/summary）
   - 每个命令显示为可点击的卡片
   - 显示命令描述
   - 点击后展开参数输入表单

2. **执行界面**
   - 动态生成参数输入框
   - 输出格式选择
   - 执行按钮
   - 结果展示区域（JSON/Table）

3. **样式优化**
   - 使用简单的 CSS 美化
   - 响应式布局
   - 加载状态提示

**实现工作量：** ~200 行 HTML/CSS/JS

### 阶段2：完整优化（可选）
**目标：完整的 Web 应用体验**

1. **高级功能**
   - 搜索和过滤
   - 收藏功能
   - 执行历史
   - 批量执行

2. **UI 框架**
   - 使用 Vue.js/React 重写前端
   - 组件化设计
   - 状态管理

**实现工作量：** ~1000+ 行代码

## 推荐实施

**立即实施：阶段1**
- 修改 server/mod.rs 中的 dashboard HTML
- 添加简单的 JavaScript 交互
- 改进 CSS 样式

**后续考虑：阶段2**
- 根据使用反馈决定是否需要

## 实施步骤

1. 修改 `dashboard_html()` 函数
2. 添加 JavaScript 执行逻辑
3. 改进 CSS 样式
4. 测试功能

## 预期效果

**改进前：**
```
apple-podcasts/search -
arxiv/paper -
...
```

**改进后：**
```
▼ HackerNews (7个命令)
  [top] 热门故事 → [执行]
  [best] 最佳故事 → [执行]
  ...

▼ Wikipedia (3个命令)
  [search] 搜索文章 → [执行]
  ...
```

点击 [执行] 后弹出参数输入表单。
